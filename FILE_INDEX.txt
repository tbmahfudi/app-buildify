  RBAC functions           → /frontend/assets/js/rbac.js

  Permission model         → /backend/app/models/permission.py
  Role model               → /backend/app/models/role.py
  Permission checking      → /backend/app/core/dependencies.py
  User model               → /backend/app/models/user.py
  Module manifests         → /frontend/modules/{name}/manifest.json

PERMISSION FORMAT:
  resource:action:scope
  Examples:
    - financial:accounts:read:company
    - users:create:tenant
    - audit:export:tenant
    - security_policy:manage:all

MENU ITEM PROPERTIES:
  title        - Display label (required)
  route        - Hash route like "dashboard" (required)
  icon         - Phosphor icon class (optional)
  roles        - Array of required roles (optional)
  permission   - Permission code required (optional)
  submenu      - Array of child items (optional)

KEY RBAC ENDPOINTS:
  GET /rbac/roles
  GET /rbac/permissions
  GET /rbac/users/{user_id}/permissions
  GET /rbac/users/{user_id}/roles
  POST /rbac/users/{user_id}/roles
  GET /rbac/organization-structure

KEY FRONTEND FUNCTIONS:
  hasPermission(code)      - Check permission
  hasRole(role)            - Check role
  filterMenuByRole(items)  - Filter menu items
  canViewField(meta)       - Field visibility
  canEditField(meta)       - Field editability

================================================================================
                         HOW TO USE THIS INDEX
================================================================================

1. For detailed documentation:
   → Read CODEBASE_ANALYSIS.md (comprehensive overview)

2. For quick lookups:
   → Use MENU_RBAC_QUICK_REFERENCE.md (cheat sheet)

3. To understand menu loading:
   → Start at /frontend/config/menu.json, then /frontend/assets/js/app.js

4. To understand RBAC:
   → Check /backend/app/models/ for models
   → Check /backend/app/routers/rbac.py for API
   → Check /frontend/assets/js/rbac.js for frontend

5. To add new permissions:
   → Create permission in database
   → Assign to roles via /rbac endpoints
   → Add checks to frontend (hasPermission)
   → Add checks to backend (has_permission dependency)

6. To modify menu:
   → Edit /frontend/config/menu.json for core menu
   → Edit manifest.json for module menus
   → Menus auto-filter at runtime based on permissions
