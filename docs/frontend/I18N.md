# Internationalization (i18n)

## Overview

App-Buildify uses **i18next** with the **i18next-http-backend** plugin to load translation files lazily. Translations are organized into namespaces per page area.

---

## Supported Languages

| Code | Language |
|------|---------|
| `en` | English (default) |
| `de` | German |
| `es` | Spanish |
| `fr` | French |
| `id` | Indonesian |

---

## File Structure

```
frontend/assets/i18n/
├── en/
│   ├── common.json     # Shared terms (Save, Cancel, Loading...)
│   ├── pages.json      # Page-specific labels
│   └── menu.json       # Navigation menu labels
├── de/
│   ├── common.json
│   ├── pages.json
│   └── menu.json
├── es/ ...
├── fr/ ...
└── id/ ...
```

---

## Translation File Format

**`common.json`**:
```json
{
  "save": "Save",
  "cancel": "Cancel",
  "delete": "Delete",
  "confirm": "Are you sure?",
  "loading": "Loading...",
  "error.generic": "An error occurred",
  "error.not_found": "Not found"
}
```

**`pages.json`**:
```json
{
  "dashboard": {
    "title": "Dashboard",
    "welcome": "Welcome, {{name}}!"
  },
  "users": {
    "title": "User Management",
    "create": "Create User",
    "table.columns.name": "Full Name",
    "table.columns.email": "Email"
  }
}
```

**`menu.json`**:
```json
{
  "dashboard": "Dashboard",
  "users": "Users",
  "rbac": "Roles & Permissions",
  "settings": "Settings"
}
```

## Initialization (`i18n.js`)

```javascript
import i18next from 'i18next'
import HttpBackend from 'i18next-http-backend'

i18next
  .use(HttpBackend)
  .init({
    lng: localStorage.getItem('language') || navigator.language.split('-')[0] || 'en',
    fallbackLng: 'en',
    ns: ['common', 'pages', 'menu'],
    defaultNS: 'common',
    backend: {
      loadPath: '/assets/i18n/{{lng}}/{{ns}}.json'
    },
    interpolation: {
      escapeValue: false
    }
  })
```

## Usage in JavaScript

```javascript
import { t } from './i18n.js'

// Using default namespace (common)
t('save')                          // "Save"
t('error.not_found')               // "Not found"

// Using specific namespace
t('pages:dashboard.title')         // "Dashboard"
t('menu:users')                    // "Users"

// With interpolation
t('pages:dashboard.welcome', { name: 'Jane' })  // "Welcome, Jane!"
```

---

## Usage in HTML Templates

```html
<button data-i18n="save"></button>
<h1 data-i18n="pages:dashboard.title"></h1>
```

After i18next initializes, call `i18next.localize(document)` or use `data-i18n` attributes which are auto-processed by the initialization hook.

---

## Language Switching

```javascript
function setLanguage(lang) {
  localStorage.setItem('language', lang)
  i18next.changeLanguage(lang).then(() => {
    // Re-render UI
    document.querySelectorAll('[data-i18n]').forEach(el => {
      el.textContent = t(el.dataset.i18n)
    })
  })
}
```

## Adding a New Language

1. Create a directory: `frontend/assets/i18n/<lang-code>/`
2. Copy translation files from `en/` and translate all values
3. Add the language code to the `supportedLngs` array in `i18n.js`
4. Add a language option to the settings dropdown in `settings.js`

---

## Adding a New Translation Key

1. Add the key to `en/` files first (source of truth)
2. Add the same key to all other language files
3. Use a consistent dot-notation key path reflecting the page/context

**Naming conventions**:
- `common.*` — global reusable terms
- `pages.<page>.<element>` — page-specific labels
- `menu.<item>` — navigation labels
- `errors.<code>` — error messages

---

## Module i18n

Modules can add their own namespaces:

```javascript
// In module initialization
i18next.addResourceBundle('en', 'financial', {
  "accounts": "Accounts",
  "invoices": "Invoices"
})
```

Use with: `t('financial:accounts')`
